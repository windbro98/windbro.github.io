1. Overview of the Review Report Format

1.1 Structure

Structure of the review varies between journals, It’s possibly informal or formal

1.1.1 Informal Structure
  • Characteristic: No criteria for reviews beyond asking for you “analysis of merits”.
  • Todo: Familiarize yourself with examples of other reviews done for the journal, which the editor should be able to provide. Or, rely on your own evolving style.
1.1.2 Formal Structure
  • Characteristic: Ask you to address specific questions via a questionnaire, or rate the paper on various attributes using a scorecard

2. The First Read-Through

When you have received the article abstract, you should already understand the aims, key data, and conclusions of the paper. If not, you need to feedback on how to improve those sections. Remember to take notes of the possible flaws or questions.

The first read-through is a skim-read. It will help you form an initial impression of the paper and get a sense of eventual recommendation, to be accept or reject.

2.1 First read considerations

Bear in mind the following questions, and form your overall impression based on them.

  • Main Question addressed by the research: Relevant and interesting
  • Topic: Original, how does it benefit the subject field compared with other published material
  • Writing: clear and easy to read
  • Consistency: Are the conclusions consistent with the evidence and arguments, do they address the main question
  • Evidence: If the author is disagreeing significantly with the current academic consensus, do they have a substantial case
  • Table or Figure: Do they aid understanding or are they superfluous

2.2 Potential Major Flaws

You should read the whole paper, identifying what to read first can save time by flagging major problems early on.

Examples:

  • Drawing a conclusion that is contradicted by the author’s own statistical or qualitative evidence.
  • The use of a discredited method
  • Ignoring a process that is known to have a strong influence on the area under study
2.2.1 What to examine
  • The sampling in analytical papers
  • The sufficient use of control experiments
  • The precision of process data
  • The regularity of sampling in time-dependent studies
  • The validity of questions, the use of a detailed methodology and data analysis being systematically
2.2.2 Major flaws in information

If methodology is less of an issue, it’s a good idea to look at the data tables, figures or images first. Especially in science research, it’s all about the information gathered. The issues include:

  • Insufficient data
  • Unclear data tables
  • Contradictory data that are not self-consistent or disagree with the conclusions
  • Confirmatory data that adds little to current understanding

2.3 Concluding the First Reading

After the first reading and the note-taking, including those major flaws you found, draft the first two paragraphs of your review. The first summarizing the research question addressed, and the second the contribution of the work.

If the journal has a prescribed reporting format, this draft will help you compose your thoughts

2.3.1 The First Paragraph

State the main question addressed by the research, and summarize the goals, approaches, and conclusions of the paper. The following should be achieved:

  • Help the editor properly contextualize the research, and add weight to your judgement
  • Show the author what key messages are conveyed to the reader, so they can be sure they are achieving what they set out to do
  • Focus on successful aspects of the paper, so the author gets a sense of what they’ve done well
2.3.2 The Second Paragraph

Provide a conceptual overview of the contribution of the research, consider:

  • premise: interesting and important
  • methods: used appropriate
  • data: support the conclusions

git常用操作

  1. git status: 查看当前仓库的状态

  3. git branch: 查看所有分支

    git branch -m [old_branch_name] [new_branch_name]: 分支重命名

    git branch -m [new_branch_name]: 当前分支重命名

    git add --all: 添加所有文件

  5. git commit -m "[message]": 提交文件，在这里表明了对分支内容的修改

  6. git push origin: 将该分支的内容上传到github的origin分支

    git push origin [branch_name]: 将该分支上传到github，成为一个新分支

  7. git pull origin: 将github的origin分支下载到本地，与当前分支合并

    git pull origin next:master: 将github的next分支下载到本地，与本地的master分支合并

  8. git reset --hard commit-id: 回滚

一. 光学绪论

  1. 光波作为信息载体的特点：波长很短，频率极高，光速极大，具有直接视觉效应

  2. 光是人类沟通宏观、微观和宇观三大世界的工具

  3. 光学的分支：

光与物质相互作用：分子光学；量子光学；光电子学

光的传播规律：几何光学；波动光学

  1. 波动光学：光的干涉，衍射和偏振；光学的时空相干性；波衍射理论；光学信息处理；全息术；光学传递函数

  2. 光的本性：物质结构及相互作用；现带标准模型；光子；传递电磁相互作用；光子静止质量（静止质量不为0 -> 真空色散，理论预测光子质量小于$10^{-46}$）

  3. 现代光学(modern optics)的提出：全息术原理的提出；光学传递函数作为象质评价标准的确立；激光器的诞生；相衬显微镜的发生。

二. 费马原理和变折射率光学

  1. 折射率：透明介质、光学物性参数。最初源于折射定律

​ $\frac{sini_2}{sini_1} = \frac{n_1}{n_2}$，n代表折射率，通常写作$n_1*sin i_1 = n_2 * sin i_2$

  1. 折射率与波长有关，同一种介质的折射率因光的波长而因，常常由Fraunhofer line来衡量一种介质的该性质。

​ 反映材料该性质的物理量为色散率 $\Delta = \frac{n_F - n_C}{n_D - 1}$

  1. 惠更斯原理：平行光进入介质后，其在第二种介质中的状况由惠更斯原理得到。

    $\frac{sin i_2}{sin i_1} = \frac{v_2}{v_1} = const$

​ $v$代表光在介质中的传播速度，故$\frac{n_1}{n_2} = \frac{v_2}{v_1}$

​ 介质的相对折射率：$n = \frac{c}{v}$，即相对于真空的折射率

  1. $n = \frac{\lambda_0}{\lambda}$，即同一条谱线的光在此介质中的波长，导出过程：

​ 因此，时间频率是光的本质频率，是眼睛实际接收到的频率

​ 以橙光为例，其在折射率较大的介质中，波长改变，但是人类看到的依然是橙光。

  1. 光程：相对于真空，光走过的路程

光程的初步意义：

  • 相位差与光程的关系：沿着波的传播方向，相位逐渐落后。$\phi(P) - \phi(Q) = -\frac{2\pi}{\lambda_0}L(QP)$

  • 时间差与光程的关系：$t_P - t_Q = \frac{L(QP)}{c}$

  1. 费马原理：光线沿光程为平稳值的路径传播

  1. 费马原理可以导出 -> 直线传播定理，反射定理，折射定理，其是几何光学三定律的理论概括

​ 费马原理的推论 -> 物象之间各条光线的光程是相等的 - 物象等光程性

  1. 理想光学系统：物空间的每一个物点对应像空间的一个像点

​ 现实中几乎没有理想光学系统（除了平面镜），但是存在对应的共轭点。下面的Q与Q’都是共轭点，也叫做齐明点（即放置光源的效果相同）
